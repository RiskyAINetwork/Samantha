How the module system works.

The module system is very easy to work with (in our opinion) for new people (people new to code
and such). The way the system works is by finding every file in a directory and loading it when
the user says something, by doing this you dont have to use any knollege or try to work out our
messy code just to add your code.

Starting your first module. Mac and Linux

1) Navigate to your home directory.
  -Mac: Cmd + Shift + G type in ~/
  -Linux: Open your favorite file browser. Navigate to ~/
2) Enter the Asher directory.
3) Enter the next asher directory.
4) Enter the plugins directory.
5) Create a new file. Name it what you want, but it has to end in _plugin.py
6) Follow the structure below.

Layout and Structure of your plugin.


The header.

At the top of every plugin, should be the header. The header should look as follows.

from asher.conversation import Statement
from time import sleep
from os import system
import random
import re
from asher.required.required import *

The core - Basic.

The core comprises of a few basics and then all your code.
Here is an example of our code.

class Plugin:
    def __init__(self):
        """
        Please leave this here or it will cause errors.
        """
        self.the_command = ['clear']

    def action(self, command, statement, ash):
        ash_print(ash, 'Clearing log.')
        system('clear')
        ash_print(ash, 'Finished')

############################
## HOW ALL THIS ^^^ WORKS.##
############################
The way it works, is by you give it the command. self.the_command = ['clear']
Then, you tell it what to do below in the def action(self, command, statement, ash):
After that, it is up to you.
If you would like to know of some of the commands you can make it do. Navigate to the next section.

The core - Advanced.

This is the advanced core. You still use the normal header above this.
This core is for multiple commands. So, if you want all your modules in the one
file, then you would do it like this.

class Plugin:
    def __init__(self):
        """
        Please leave this here or it will cause errors.
        """

        self.who = ['who are you', '?']
        self.help = ['help']
        self.exit = ['exit please', "quit"]

        self.the_command = self.who + self.help + self.exit

    def action(self, command, statement, ash):

        if command in self.who:
            ash_print(ash, 'My name is Andrew. I am here to help and serve you.')
            sleep(1)
            ash_print(ash, 'Feel free to ask me anything. If i do not know it, I will learn it.')
        elif command in self.help:
            ash_print(ash, 'what with?')
        elif command in self.exit:
            ash_print(ash, 'Good bye sir.')
            exit()

############################
## HOW ALL THIS ^^^ WORKS.##
############################
Everything here is pretty standard. The main thing that changes is the grabbing commands.
To take a command you must do it like: self.something = ['something']
(replaceing "something" with your code.)
Also, if you have multiple commands, you have to add them to. self.the_comand = self.something
the self.something would be what ever you defined above. To add more commands, use a +
Now, in what the commands actually do.
If you have more than one, you will have to use if and elif statements.
it works the same way as before, just you define what the command is first.
eg. if command in self.something: or elif command in self.something:
then, anything after that is whatever you want.

All the commands you can or have to use.


- To make Asher say something: ash_print(ash, 'your message here')
- To use a terminal command: system('your command here')
- To make Asher wait: sleep(time here in seconds) eg. sleep(2)   - 2 seconds
- Make Asher require a word for a command: self.the_command = ['your word']
- Make Asher require more words for a command: self.the_command = ['word 1', 'word 2', 'word 3']
...As development goes on, more will be added...
